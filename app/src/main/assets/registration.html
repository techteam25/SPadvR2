General: 

  1. The Registration screen is generally a one-off activity that is done during the first part of a training workshop or when a local team first begins to use the StoryProducer (SP) app.

  2. Tap each information “drawer” to expand it and type in the information. Scroll down to access each field.  Tap on a line to place the cursor and bring up the keyboard ready to type information. A few fields are drop down menus which you need to tap on and choose an item. Tap the “drawer” again to close it.

  3. If you SKIP REGISTRATION, you can go directly to the Story Templates list. The next time you open the StoryProducer app, the Registration screen will appear. If youSUBMIT the Registration, the Registration screen will be hidden and not appear each time you open the app.

  4. If the Registration information needs to be edited and (re)submitted, it can be accessed by tapping on the top left 3-bar menu icon in any screen which will open a sidebar menu. Tap on Update Registration.

  5. Filling all the fields and submitting the Registration is not a requirement to use the app, but it is helpful to submit the Registration information to assist with future analytics and troubleshooting.

  6. It is suggested that the local SP team leader input the information requested with the help of the SP Trainer. Contact information will need to be provided for the primary translators, the Accuracy checker and the Trainer. The Trainer should be prepared with language information readily available.

  7. The SP Trainer is ultimately responsible for submitting the Registration info via email to SPapp.training@gmail.com and to SPapp_info@sil.org. When submitting via email, all other emails listed in the Registration fields will automatically be inserted into the email addressee section.

  • If a local SP team phone does not have internet connectivity, the Registration info (a small text doc) can be submitted via blue-tooth to the Trainer’s phone. Then when the Trainer gets online, they can resubmit or forward the Registration info on to SPapp.training@gmail.com and to SPapp_info@sil.org.

Language Information:

  • The Ethnologue Code is a 3-letter code which has been assigned to each language. Tap on the WHAT IS THIS? link to access the online Ethnologue database to search for a specific code. If you are working in a language or dialect that has not yet been assigned a code, please type the 3-letter code of the most closely related language followed by a dash (no spaces) followed immediately by 3 primary letters of the dialect name. For example, PEX-MAJ. This code, if inserted and submitted, will appear in the partially automated file name of all videos produced so that it is clear with which language the video is produced.

  • Orthography Status has to do with the newness and stability of the language writing system or alphabet. Tap on the drop-down menu and then tap on the most appropriate option.

Translator Information:

  • If you wish to enter information for more than one translator, simply put a comma or a slash and list the multiple people with their corresponding information on the same line.

Consultant Information:

  • This drawer or section is where information is registered for the Accuracy Checker. The Consultant (also called the Accuracy Checker) should not be the same person as a Translator. The Consultant or Accuracy Checker may or may not be the same person as the Trainer.

    • Ideally, the Accuracy Checker for Bible stories should be a Translation or SE Consultant.  Certainly, an Accuracy Checker should be someone with translation training and years of experience in translating the Bible, involvement in checking processes, sorting out key terms in a thoughtful way and access to exegetical helps.

  • It is important for the Trainer to ensure that every StoryProducer language project has been assigned and connected to an Accuracy Checker who can assist with approving each story. Only the Accuracy Checker is entrusted with an approval password by the Trainer.

  • A Consultant (Accuracy Checker) will be doing in-person accuracy checks on each story, personally (or by proxy e.g. via Whatsapp) holding the SP phone, and controlling the check and any revisions required before approving the story for publication and distribution.  Usually a local Accuracy Checker speaks the local language.

Trainer Information:

  • This drawer or section contains the contact information for the certified SP Trainer who is helping a new StoryProducer language team get set up.

    • People get qualified to be SP Trainers by taking a course or self-educating (by working through the unlocked demo story available from the main menu, reading all the Help docs in each phase, perhaps watching some training videos) and passing the “Qualify as a SPadv Trainer” Quiz. Contact SPapp.training@gmail.com for training options and for access to the SPadv Trainer Quiz.

    • SP Trainers are responsible for:

      • setting up and training local language teams in use of the StoryProducerAdv app

      • connecting local SP teams with a local or remote Accuracy Checker

      • training and mentoring other SP Trainers

      • sharing the SP approval password only with Accuracy Checkers

      • submitting Registration information (via email) for teams that they train

    • Usually, about a week of on-site or remote training is required from a certified StoryProducer Trainer to get a local language team set up and running competently on the SPadv app.

  • The Trainer’s location should be the place where the Trainer normally resides most of the time.

Archive Information:

  • This is where other email addresses can be typed if there is an organization or interested party who wants to receive the Registration information.

  • If an organization or interested party would like to receive produced videos, a whatsapp number or an email address could be entered here to provide a way for the local SP team leader to share videos with them.
